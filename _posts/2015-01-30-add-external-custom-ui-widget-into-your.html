Basically each widget is an Android application and you can actually import it into your own app to make your app look better. I want to try the Circular Progress Button which is a well made button widget to indicate the progress of a certain time consuming process if you click it. It's easy to use it, what you need to do is to add a tag with the name of the widget in your layout xml file. And the block of widget in xml looks like this:
<com.dd.CircularProgressButton
    android:id="@+id/btnWithText"
    android:layout_width="196dp"
    android:layout_height="64dp"
    android:layout_marginTop="16dp"
    android:textColor="@color/white"
    android:textSize="18sp"
    app:cpb_textComplete="@string/Complete"
    app:cpb_textError="@string/Error"
    app:cpb_textIdle="@string/Upload" />
The problem is, however, how to import the project that contains the code of view building into your project. Normally we configure the Java Build Path and include the external library and project to make it work. When I try to include the project into the build path, it fails to recognize the external widget class. After trying different ways, I happened to find that there is a Is Library checkbox in the Properties window under the Android tab and there is a Add button on the right. I thought maybe I should click the checkbox and then import it as a library like it said in my own project. So I did select the checkbox in the external widget project like the following.
